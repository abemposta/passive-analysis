About {{ protocol }}

This protocol can give information such as:

  • Manufacturer: The first bits of the MAC of the devices are related to the device manufacturer. If the user did not change its MAC, the MAC address itself will reveal the device manufacturer.
  • IP address: This protocol uses the IP layer, so you can get the device IP address.
  • Operating System: This protocol uses the IP protocol. OSs use different TTL. For example, linux might use a TTL of 64 meanwhile windows might use a TTL of 128.
  • Windows Version: The protocol BROWSER contains a parameter windows_version that contains de windows version of the device.
  • Type of device: The protocol BROWSER contains a parameter server_type that indicates diverse information about the device type.
  • Application port: This protocol uses the UDP protocol. The value srcport of the UDP protocol contains the port used by the application.
  • Hostname: The NBDGM protocol used by {{ protocol }} contains the value source_name that most likely matches with the hostname of the device.
  • Workgroup: The NBDGM protocol used by {{ protocol }} contains the value destination_name that contains the workgroup of the device.
  • Extra information: The field comment of the BROWSER layer often contains extra information about the device or its role.
